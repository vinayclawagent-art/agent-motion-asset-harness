## Trial flow
1. Fill `Lottie Asset Brief Builder.html` during a real VinClawLabs motion-asset task.
2. Fill `Trial Evidence Index.md` with the generated Lottie/JSON, preview screenshot or recording, design-review notes, and handoff diff/spec links.
3. Fill `Post-Trial Promotion Decision Card.md` only after indexed evidence exists.
4. Complete `Post-Trial Motion Debrief Template.md` to decide exactly which README, prototype, infographic, or skill-draft claims should change.
5. Patch the package README, skill draft, or productization plan only from indexed/debrief evidence.
